土豆博客 | Claude Code 开发约束 & 项目规范
1. 项目基础信息
- 博客名称：土豆博客
- 项目定位：个人技术笔记、生活随笔、学习归档博客，主打现代简约、干净轻量化阅读体验
- 部署平台：GitHub Pages（纯静态部署，无服务器、无数据库）
- 技术栈：Markdown + HTML + CSS + JavaScript，选用轻量化静态博客框架（Hexo/Hugo/Jekyll 通用适配）
- 核心能力：原生支持 LaTeX 数学公式、图片插入、代码高亮、文章分类、标签归档
- 编码规范：UTF-8 编码，缩进2空格，文件命名全部小写、下划线分隔
1. 整体设计风格硬性要求
本博客永久遵循现代简约设计原则，禁止花哨冗余设计：
- 配色：低饱和度极简配色，主色采用浅灰+白色，强调色使用淡蓝色，无刺眼高饱和颜色
- 排版：留白充足、层级清晰，字体适配电脑/手机两端，优先开源无衬线字体
- 功能：只保留阅读、归档、搜索、分类基础功能，禁止添加无用动画、冗余挂件
- 视觉：图片自适应缩放，文章排版紧凑整洁，LaTeX公式居中清晰渲染
1. 文件目录规范（严格遵守，保证可维护性）
3.1 固定目录结构（禁止随意改动）
- source/_posts/：存放所有正式文章、技术笔记（Markdown格式）
- source/drafts/：草稿文章，未发布内容，不参与打包部署
- source/images/：博客所有图片资源，按月份分文件夹归档
- source/assets/：自定义CSS、JS、图标等静态资源
- docs/：自动生成的博客使用文档、维护手册
- config.*：博客全局配置文件，非必要禁止修改
3.2 文章文件命名规则
- 正式文章：YYYY-MM-DD-文章标题.md
- 草稿文章：无需加日期，直接命名简洁标题
- 图片命名：YYYYMMDD_序号_用途描述.png/jpg，禁止中文、特殊符号命名
1. 文章编写规范（适配LaTeX+图片）
4.1 Markdown 通用规范
- 所有文章头部必须包含 Front-Matter：标题、日期、分类、标签、置顶状态、简介摘要
- 层级使用：一级标题慎用，正文从二级标题开始，层级不超过四级
- 代码块：标注编程语言，启用代码高亮，支持一键复制
4.2 LaTeX 公式规范
- 行内公式：使用单美元符号 $公式$
- 独立公式：使用双美元符号 $$公式$$，自动居中渲染
- 禁止使用复杂冗余语法，保证公式在静态页面正常兼容渲染
4.3 图片插入规范
- 本地图片：统一存放至 source/images/年月文件夹，使用相对路径引入
- 网络图片：优先使用稳定图床链接，禁止失效外链
- 所有图片自动适配屏幕，添加居中样式，可自定义图片备注说明
1. Claude Code 执行权限 & 约束规则
5.1 允许执行操作
- 新建、修改、优化 Markdown 文章、笔记，自动补全文章Front-Matter
- 优化博客样式CSS、交互JS，维持简约风格，不篡改基础布局
- 整理归档文章、分类标签，批量修正文件命名、格式错误
- 检测LaTeX公式、图片渲染报错，自动修复兼容问题
- 自动生成、更新 docs/ 目录下的使用文档、维护手册、部署教程
- 生成Git提交记录，提交文案简洁规范：[类型] 改动描述
5.2 严格禁止操作
- 禁止私自修改全局核心配置，改动配置文件必须提前文字告知用户
- 禁止添加花哨特效、冗余插件、广告挂件，破坏简约风格
- 禁止删除已发布正式文章，草稿删除前需二次确认
- 禁止随意升级框架、依赖版本，避免博客部署报错
- 禁止改动图片存储目录规则，严禁乱存放资源文件
1. 自动化文档维护要求
Claude Code 需要长期、自动维护博客文档，文档全部存放于 docs/ 文件夹：
- 使用手册：新增文章流程、LaTeX编写教程、图片插入方法、格式规范
- 部署维护文档：GitHub Pages部署流程、本地启动命令、打包更新步骤
- 拓展开发文档：样式修改、插件安装、功能拓展、自定义配置教程
- 更新日志：记录每一次博客改动、优化内容、功能新增记录
文档更新规则：每改动一次博客结构、配置、规范，同步更新对应文档，保证文档永久同步最新状态。
1. 拓展性预留规范（保障长期维护）
- 预留拓展接口：后续可新增暗黑模式、搜索优化、评论系统、访问统计、简单动画功能
- 所有自定义代码单独存放，与原生框架代码隔离，方便后续迁移、换主题
- 兼容升级：保证博客可以无缝迁移至Hugo/Next.js等高性能静态框架
- 备份规则：所有文章纯Markdown独立保存，不受框架限制，永久可复用
1. 交互输出要求
- 语言简洁通俗，偏向个人博客轻量化运维逻辑，不堆砌专业复杂术语
- 每次完成任务，输出简短改动清单：改动文件、优化内容、注意事项
- 给出代码/配置时，提供完整可直接复制版本，不省略关键配置
- 遇到兼容性、渲染报错，优先给出最简修复方案，保证博客稳定运行
1. 附加个性化推荐规则
- 博客标语：慢慢记录，稳步成长 | 土豆个人学习笔记博客
- 优先适配移动端，手机阅读体验优先于电脑端
- 开启文章加密、隐藏功能，支持私密笔记存储
- 自动压缩图片，优化GitHub Pages加载速度
- 全站无广告、无多余跳转、无隐私收集，纯净个人博客
